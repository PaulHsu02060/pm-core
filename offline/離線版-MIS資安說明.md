# PM-Core 離線版 — 資安說明（給 MIS / 資訊部門）

> 一頁講清楚：這個工具是什麼、資料流向、如何自行驗證。
> 檔案：`pm-core-offline.html`（單一 HTML 檔·自帶全部程式與字型·約 4MB）

## 這是什麼

專案管理個人工具（排程／進度／負荷計算／報表）。**純前端網頁單檔**：用瀏覽器開啟（雙擊或 `file://`），不安裝任何軟體、不需要管理員權限、不建立任何服務或排程工作。

## 資料流向（核心）

| 問題 | 答案 |
|---|---|
| 會連外網嗎？ | **不會。零對外連線。** 所有程式庫、字型、圖示全部內嵌在單檔內；雲端同步／帳號登入／AI 功能在此版本已整組移除。 |
| 資料存在哪？ | **只存在這台電腦的瀏覽器 localStorage**。不上傳、不同步、不落地到任何伺服器。 |
| 備份怎麼做？ | 使用者手動「下載 JSON 備份」存到公司核准的位置（公司硬碟／公司雲端）。檔案不離開公司環境。 |
| 需要帳號嗎？ | 不需要。無登入、無外部身分驗證。 |
| 會自動更新嗎？ | 不會。更新＝人工替換新的單檔（同樣流程再審一次即可）。 |

## MIS 如何自行驗證（兩分鐘）

1. 用瀏覽器開啟 `pm-core-offline.html`。
2. 按 `F12` → 切到 **Network（網路）** 分頁 → 重新整理頁面。
3. 確認清單中**沒有任何 `http(s)://` 對外請求**（只有 `file://` 本身）。
4. 可再實際操作幾個功能（建任務、開報表）——Network 持續為零。

這不是口頭承諾：單檔內沒有任何 `<script src="http…">` / `<link href="http…">` 外部資源引用（打包流程有自動防呆，殘留任何外部引用即中止產出），對外請求「結構上」不存在。

## 補充

- **來源與版本**：檔案開頭註解帶有 build 版號（日期＋版本代碼），可核對目前使用版本。
- **程式碼**：為保護智慧財產已壓縮（去空白註解），但行為可由上述 Network 驗證完整確認。
- **移除乾淨**：不想用時，刪除 HTML 檔＋清除該瀏覽器分頁的網站資料即可，系統不留任何其他痕跡。
